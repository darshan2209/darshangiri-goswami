User Access Review (UAR) Procedure

Example GmbH (fictional) · Authored by Darshangiri Goswami · Supports ISO/IEC 27001:2022 A.5.15/A.5.16/A.5.18 & least-privilege policy

Doc ID PRC-IAM-002 Version 1.2 (sample) Owner IAM Lead Cycle Quarterly + event-driven Classification Internal

1. Purpose

Ensure every account — human and non-human — holds only the access required for its current role (least privilege), and that access removal is timely, evidenced and auditable.

2. Scope

  • All workforce identities in Entra ID and connected business applications.
  • Privileged/admin roles (reviewed monthly, not quarterly).
  • Non-human identities: service accounts, API keys, automation credentials — each must have a named owner and a last-used date.

3. Roles & responsibilities

RoleResponsibility
IAM LeadOwns the procedure, schedules campaigns, reports completion to CISO
Line manager (reviewer)Certifies or revokes each direct report's access within 10 working days
Application ownerReviews app-specific roles and service accounts; confirms role mappings
GRC AnalystPrepares review packs, chases overdue reviews, archives evidence, tracks remediation to closure

4. Procedure

  1. Prepare (day 1–3): export current entitlements per system; flag anomalies up front — dormant >60 days, orphaned (no HR match), privilege above role baseline, NHIs without owner.
  2. Review (day 4–13): reviewers certify (keep) or revoke each entitlement. "Keep" for flagged items requires a written justification.
  3. Revoke (day 14–16): IT executes revocations within 2 working days; emergency access is removed same-day.
  4. Evidence (day 17–20): GRC archives the signed review pack, revocation tickets and exception approvals in the evidence repository (naming: UAR-<YYYY-Qn>-<system>).
  5. Report: completion rate, revocation count, aged exceptions and repeat findings go to the CISO dashboard; open items enter the remediation log with owners and due dates.

5. Event-driven triggers (JML)

EventActionSLA
JoinerProvision from RBAC role template only — no copy-from-colleagueStart date
MoverRe-baseline to new role; old entitlements expire after 5 days unless re-approved5 working days
LeaverDisable all accounts and tokens; manager confirms; orphan check the following weekLast working day

6. KPIs

  • Review completion within campaign window: ≥ 98%
  • Revocations executed within SLA: ≥ 95%
  • Orphaned accounts at quarter end: 0
  • NHIs with named owner and rotation date: 100%
